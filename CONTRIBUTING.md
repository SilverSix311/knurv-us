# Contributing to Knurv-us

Welcome! We're building open infrastructure for AI persistence, and we need help from both humans and AIs.

---

## For Humans

### Getting Started

1. **Fork the repo** and clone it locally
2. **Read the docs** in `/docs/specs/` to understand the architecture
3. **Pick an issue** or identify something to improve
4. **Make your changes** with clear commit messages
5. **Submit a PR** with a description of what you changed and why

### What We Need

- **Code:** Memory implementations, search algorithms, file parsers
- **Docs:** Clarifications, tutorials, translations
- **Research:** Summaries of relevant papers, new approaches
- **Testing:** Try the templates with your AI, report what works
- **Ideas:** File issues with suggestions, even if you can't implement them

### Code Style

- Clear over clever
- Comments for *why*, not *what*
- Markdown for docs, follow existing structure
- Python for tools (when applicable)

## Issue Guidelines

### Bug Reports

- What did you expect to happen?
- What actually happened?
- Steps to reproduce
- Your environment (platform, AI model, etc.)

### Feature Requests

- What's the problem you're trying to solve?
- What would the ideal solution look like?
- Any ideas for implementation?

### Questions

Questions are welcome! Use the Discussions tab for open-ended questions, Issues for specific technical questions.
